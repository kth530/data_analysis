# RFM 세그먼테이션 방법론

`notebooks/03_rfm_segmentation.ipynb`에서 사용한 RFM 점수화·가중치·등급 배정 방법을 정리한다.

## 1. RFM 지표 산출

고객ID 단위로 세 지표를 집계한다 (기준일 `2019-12-31`).

| 지표 | 정의 |
|------|------|
| Recency | 기준일 − 마지막 구매일 (작을수록 우수) |
| Frequency | 구매 거래 건수 |
| Monetary | 총 구매금액(세후) |

## 2. 점수화 (1-5점)

지표마다 분포 특성이 달라 점수화 방식을 다르게 적용한다.

- **R — Quantile 5등분**: Recency 분포가 비교적 균등해 단순 5분위(`qcut`)를 적용한다. 최근일수록 높은 점수(역순).
- **F / M — IQR 기반 5등분**: 두 지표 모두 우편향(right-skewed)으로 이상치가 존재한다. 단순 5등분 시 이상치가 일반 고객과 같은 구간에 묶이는 문제가 있어, IQR 상한(`Q3 + 1.5×IQR`)을 기준으로 이상치를 5점 구간으로 분리한다.

  | 점수 | 구간 |
  |------|------|
  | 1 | ≤ Q1 |
  | 2 | Q1 – Q2 |
  | 3 | Q2 – Q3 |
  | 4 | Q3 – (Q3 + 1.5×IQR) |
  | 5 | Q3 + 1.5×IQR 초과 (이상치) |

> **척도 비대칭 주의**: R은 분위 기반이라 점수가 5구간에 고르게 퍼지지만, F/M은 우편향이라 대부분 저점에 몰린다. 두 방식의 분포 폭이 다르므로 합산 점수에서 R의 변동 기여가 상대적으로 커진다. 아래 PCA 가중치는 이 분포 구조를 반영한 결과다.

## 3. 가중치 — PCA 기반

세 점수를 하나로 합산할 때 가중치를 정하는 방법:

- **동일 가중치(1/3)**: 세 지표가 고객 구분에 동등하게 기여한다는 가정 — 데이터 분포 차이를 무시한다.
- **수작업 조정**: 분석가 직관에 의존 — 주관 개입.
- **PCA (사용)**: R/F/M 점수에 PCA를 적용해 첫 주성분(PC1) 로딩의 절댓값을 정규화하여 가중치로 사용한다. 고객 간 차이를 설명하는 방향을 데이터에서 도출한다.

`RobustScaler`로 표준화 후 `PCA(n_components=1)` 적용 결과:

| 지표 | 가중치 |
|------|--------|
| R | 0.201 |
| F | 0.339 |
| M | 0.460 |

- PC1 설명 분산 비율: **0.694** (세 지표 변동의 약 69%를 단일 축으로 설명)
- Monetary 가중치가 가장 높아, 고객 가치 구분에서 구매금액이 가장 큰 설명력을 가진다.

## 4. 종합 점수

```
RFM Score = (R × w_r + F × w_f + M × w_m) / 5 × 100   →  20–100점
```

R/F/M이 모두 1-5점이므로 최솟값 20점(전부 1점), 최댓값 100점(전부 5점)이다.

## 5. 등급 배정

가중치는 PCA로 객관화하되, 등급 경계는 운영 해석이 쉬운 **고정 점수 임계값**을 사용한다. 임계값은 점수 분포의 분위 구간과 대조해 자연스러운 경계 근처(예: Silver 50점 ≈ 중앙값 52점, Platinum 80점 ≈ 상위 ~13%, Diamond 95점 ≈ 상위 ~4%)에 맞췄다.

| 등급 | 기준 | 분포상 위치(근사) |
|------|------|------------------|
| Diamond | ≥ 95 | 상위 약 4% |
| Platinum | ≥ 80 | 상위 약 15% |
| Gold | ≥ 65 | 상위 약 32% |
| Silver | ≥ 50 | 중앙값 부근 |
| Bronze | < 50 | 하위권 |

> 분위 자동 분할(예: `qcut(5)`) 대신 고정 임계값을 쓴 이유는, 등급 정의를 데이터가 갱신돼도 일관되게 유지하고 비즈니스 커뮤니케이션에서 "80점 이상 = Platinum"처럼 해석을 단순화하기 위함이다.

## 6. KMeans 클러스터링의 위치

`KMeans(n_clusters=4)`는 **탐색·검증용**이다. RFM 점수 구조가 자연스러운 고객 군집으로 갈라지는지 확인하는 보조 단계이며, 최종 등급 배정에는 사용하지 않는다. 최종 등급은 5절의 점수 임계값으로 결정하고, 후속 등급별 분석(`04~07`)은 모두 `rfm_result.등급`을 기준으로 로드한다.
